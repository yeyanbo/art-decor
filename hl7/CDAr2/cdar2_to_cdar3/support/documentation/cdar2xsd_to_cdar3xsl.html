Documentation

Description

Created on: Sep 25, 2012

Author: Alexander Henket, Nictiz, The Netherlands

Purpose: Builds an XSL 2.0 transform to convert any valid CDAr2 instance document to a valid CDAr3 instance document. To (re)create the transform ../cda2_to_cdar3.xsl, run: 

                java -jar saxon9.jar -xsl:cdar2xsd_to_cdar3xsl.xsl -s:cdar2xsd_to_cdar3xsl.xsl

History:

  • 2013-05-14
    • Updated for May 2013 draft for comment of CDAr3
    • Rewritten logic to obtain datatyped elements by using RIM coreMif to get the list of valid attributes for a given class, also allowing for correct order processing
    • Rewritten so the datatype transforms are now available as an include, and handle namespace conversion if necessary
    • Patch for RegionOfInterest.value which was not handled due to its exceptional nature
  • 2013-03-07
    • Added codeSystem to conversions from CS to CD (languageCode, signatureCode, RegionOfInterest.code)
